Identificação do Usuário

Para a área de identificação do usuário criaremos os campos Nome, Data de Nascimento, Email, Sexo e Senha

Começaremos criando um parágrafo para cada um deles

<form>
        <fieldset>
            <legend>Identificação do Usuário</legend>
            <p>Nome:</p>
            <p>Data de Nascimento:</p>
            <p>Email:</p>
            <p>Sexo:</p>
            <p>Senha:</p>
        </fieldset>
        <fieldset>
            <legend>Endereço do Usuário</legend>
        </fieldset>
</form>

Agora vamos criar os campos onde o usuário poderá digitar as informações pedidas. Para a maioria deles usaremos a tag <input> com o parâmetro type definindo o tipo de campo que desejamos.

Vamos definir a caixa de texto para o nome:

<form>
        <fieldset>
            <legend>Identificação do Usuário</legend>
            <p>Nome:
                <input type="text" name="tNome" id="cNome">
            </p>
            <p>Data de Nascimento:</p>
            <p>Email:</p>
						<p>Sexo:</p>
            <p>Senha:</p>
        </fieldset>
        <fieldset>
            <legend>Endereço do Usuário</legend>
        </fieldset>
</form>

Como nome é texto, utilizamos o type text. Os parâmetros name e id são importantes para conseguirmos manusear esses campos através do back-end, que você ainda vai aprender.

Alguns outros parâmetros também importantes são

  • size →define quantos caracteres aparecerão simultaneamente no campo
  • maxlength →define a quantidade máxima de caracteres aceitos no campo
  • placeholder →escrita dentro do campo, espécie de dica para o usuário
<form>
    <fieldset>
        <legend>Identificação do Usuário</legend>
        <p>Nome:
            <input type="text" name="tNome" id="cNome" size="30" maxlength="200" placeholder="Nome completo">
        </p>
        <p>Data de Nascimento:</p>
        <p>Email:</p>
				<p>Sexo:</p>
        <p>Senha:</p>
    </fieldset>
    <fieldset>
        <legend>Endereço do Usuário</legend>
    </fieldset>
</form>

Teste isso no navegador e veja se entendeu todos os conceitos para partirmos pros próximos campos

Dependendo do navegador que você estiver usando, até um calendário para facilitar a seleção já aparecerá

Para o email, novamente usaremos input, agora com o type email. Também definiremos size e maxlength. Esse tipo de campo serve para o navegador validar se o email está no formato certo (digitado com @ e ponto)

<form>
    <fieldset>
        <legend>Identificação do Usuário</legend>
        <p>Nome:
            <input type="text" name="tNome" id="cNome" size="30" maxlength="200"
            placeholder="Nome completo">
        </p>
        <p>Data de Nascimento:
            <input type="date" name="tData" id="cData">
        </p>
        <p>Email:
           <input type="email" name="tEmail" id="cEmail" size="30" maxlength="40">
        </p>
        <p>Sexo:</p>
        <p>Senha:</p>
    </fieldset>
    <fieldset>
        <legend>Endereço do Usuário</legend>
    </fieldset>

Para o sexo, vamos fazer aquelas bolinhas pra selecionar uma das duas opções, conhecidas em html como radio buttons.

Elas também serão criadas com input, dessa vez com type radio, e sempre que pertencerem à um mesmo grupo de seleção, devem ter o mesmo name

<form>
    <fieldset>
        <legend>Identificação do Usuário</legend>
        <p>Nome:
            <input type="text" name="tNome" id="cNome" size="30" maxlength="200"
            placeholder="Nome completo">
        </p>
        <p>Data de Nascimento:
            <input type="date" name="tData" id="cData">
        </p>
        <p>Email:
           <input type="email" name="tEmail" id="cEmail" size="30" maxlength="40">
        </p>
        <p>Sexo:
            <input type="radio" name="tSexo" id="cMasc">Masculino
            <input type="radio" name="tSexo" id="cFem">Feminino
         </p>
        <p>Senha:</p>
    </fieldset>
    <fieldset>
        <legend>Endereço do Usuário</legend>
    </fieldset>
</form>

Observe que dessa forma ele só permitirá que você selecione uma das opções

Mas temos um problema, você tem que acertar exatamente dentro da bolinha pra ela ser selecionada, o que não é muito legal. Para melhorar, podemos fazer com que ao clicarmos em "Masculino" ou "Feminino" os seus campos sejam selecionados

Para isso, utilizaremos a tag <label></label> (etiqueta) fazendo uma ligação com esses campos através do seu id. Assim:

<form>
  <fieldset>
    <legend>Identificação do Usuário</legend>
    <p>Nome:
        <input type="text" name="tNome" id="cNome" size="30" maxlength="200"
        placeholder="Nome completo">
    </p>
    <p>Data de Nascimento:
        <input type="date" name="tData" id="cData">
    </p>
    <p>Email:
       <input type="email" name="tEmail" id="cEmail" size="30" maxlength="40">
    </p>
   <p>Sexo:
        <input type="radio" name="tSexo" id="cMasc"><label for="cMasc">Masculino</label> 
        <input type="radio" name="tSexo" id="cFem"><label for="cFem">Feminino</label>
        </p>
    <p>Senha:</p>
  </fieldset>
  <fieldset>
    <legend>Endereço do Usuário</legend>
  </fieldset>
</form>

Teste para ver se quando clica nas palavras os radio buttons são selecionados.

Você pode fazer isso com qualquer tipo de input, não apenas radio

Para o campo de senha usaremos um input com type password

Assim como nos outros campos, podemos definir size, maxlength e placeholder

<form>
  <fieldset>
    <legend>Identificação do Usuário</legend>
    <p>Nome:
        <input type="text" name="tNome" id="cNome" size="30" maxlength="200"
        placeholder="Nome completo">
    </p>
    <p>Data de Nascimento:
        <input type="date" name="tData" id="cData">
    </p>
    <p>Email:
        <input type="email"
        name="tEmail" id="cEmail" size="30" maxlength="40">
    </p>
    <p>Sexo:
        <input type="radio" name="tSexo" id="cMasc"><label for="cMasc">Masculino</label> 
        <input type="radio" name="tSexo" id="cFem"><label for="cFem">Feminino</label>
    </p>
    <p>Senha:
        <input type="password" name="tSenha" id="cSenha" size="8"
        maxlength="8"
        placeholder="8 dígitos">
    </p>
  </fieldset>
  <fieldset>
    <legend>Endereço do Usuário</legend>
  </fieldset>
  </form>

Visualmente ela é igual à caixa de texto, mas experimente digitar algo nela e verá a diferença
